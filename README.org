* FastCode Native

A native desktop client for [[https://opencode.ai][OpenCode Zen]], built in C++ with wxWidgets.

No Electron. No Chromium. No 800 MB RAM for a chat window.

#+begin_quote
/Screenshots and demo GIF coming soon -- the app runs today on Linux, macOS, and Windows./
#+end_quote

** The Problem

Every AI coding tool ships as an Electron app or a web-based TUI. You're running a compiler, a language server, a browser with 47 tabs, and then your AI assistant wants another Chromium instance and half a gigabyte of RAM to show you a text conversation.

Native desktop apps solve this, but nobody builds them anymore because it's hard. FastCode Native is an attempt to change that.

** What It Does

- Connects to *OpenCode Zen* (free models available, no API key needed)
- Renders conversations with a *custom text control* -- not a web view, not a rich text widget, a purpose-built component with pixel-perfect selection
- Runs on *Windows, macOS, and Linux* from one codebase
- Builds to a *single 9 MB binary* (stripped release) with zero runtime dependencies

** Features

- *Custom text rendering engine* -- Block-type styling for user prompts, AI responses, code blocks, and thinking/reasoning. Word wrap, Unicode, bidirectional text (Arabic, Hebrew), double/triple-click selection, drag-to-select with auto-scroll.
- *OpenCode Zen integration* -- Free anonymous access to models like Big Pickle, MiMo, Nemotron, MiniMax. API key unlocks all models.
- *OS theme integration* -- Follows system dark/light mode automatically
- *Secure API key storage* -- Uses your OS keychain (libsecret / Keychain / Credential Manager)
- *MCP Protocol server* -- Built-in Model Context Protocol support over stdio
- *Loading animations* -- Animated dots while the model is thinking

** Quick Start

#+begin_src bash
git clone https://github.com/lszl84/fastcode-native.git
cd fastcode-native

cmake -B build -G Ninja
cmake --build build

./build/fastcode-native
#+end_src

No =npm install=. No =node_modules=. CMake fetches all dependencies (wxWidgets, nlohmann/json) automatically.

** How It's Built

Flat source layout, every file in the project root:

| File                       | Purpose                              |
|----------------------------+--------------------------------------|
| =streaming_text_ctrl.cpp/h= | Custom text control (the core)       |
| =block_manager.cpp/h=       | Content block storage                |
| =main_frame.cpp/h=          | Main window and UI                   |
| =http_client.cpp/h=         | Async HTTP via wxWebRequest          |
| =zen_client.cpp/h=          | OpenCode Zen API client              |
| =mcp_server.cpp/h=          | MCP protocol server (stdio JSON-RPC) |
| =application.cpp/h=         | wxApp entry point                    |

The =StreamingTextCtrl= is the interesting part. It's a fully custom =wxControl= with:

- O(log N) block lookup via prefix-sum cache
- Lazy per-character caret position arrays (only computed for lines involved in selection)
- Word-wrap with hard and soft line breaks
- Pixel-perfect selection across mixed LTR/RTL content
- Incremental layout -- only re-wraps blocks that changed

The text control is adapted from [[https://github.com/lszl84/cutescroll][cutescroll]].
